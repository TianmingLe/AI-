# GPS 模块与 UDP 局域网发现协议集成设计

## 1. 概述
在 AI 眼镜的固件系统中，我们计划新增 **GPS/GNSS 传感器** 以提供实时地理位置服务（如户外导航、轨迹记录），并新增 **UDP 局域网广播协议**，以便于在同一局域网下的手机 App 或管理终端能够自动发现并连接 AI 眼镜。

## 2. 硬件与协议设计

### 2.1 GPS/GNSS 传感器 (GpsSensor)
- **接口类型**: 串口 (UART)。通常在 `/dev/ttyS0` 或 `/dev/ttyUSB0` 上读取 NMEA 0183 格式的数据。
- **数据结构**: 包含经度、纬度、海拔和卫星数量。
- **Mock 策略**: 如果沙盒环境中无法访问真实的 UART 设备，则回退到 Mock 模式，使用模拟的 GPS 坐标（如围绕某个起点进行圆周运动的虚拟轨迹）。

### 2.2 UDP 局域网发现协议 (UdpBroadcaster)
- **底层实现**: POSIX Sockets (UDP `SOCK_DGRAM`)，启用 `SO_BROADCAST` 选项。
- **功能描述**: 眼镜作为 UDP 客户端，定期（例如每 5 秒）向局域网广播地址（例如 `255.255.255.255:9999`）发送包含设备名称、MAC 地址、IP 地址及电量等信息的 JSON 数据包。
- **应用场景**: 辅助设备（如配套的手机应用）监听对应的 UDP 端口，实现“零配置”一键连接。

## 3. 实现计划
1. **任务 1**：创建 `GpsSensor.h` 和 `GpsSensor.cpp`，实现 UART 读取与 NMEA 解析（或者 Mock 轨迹）。
2. **任务 2**：创建 `UdpBroadcaster.h` 和 `UdpBroadcaster.cpp`，实现 UDP 广播的发送逻辑。
3. **任务 3**：在 `main.cpp` 中实例化这两个组件，并将其集成到现有的传感器线程与事件总线中。
4. **任务 4**：编写相应的单元测试并在沙盒环境中进行编译与验证。
